# UC18 - Basemap Switch (Satellite / Streets)

## Status
Implemented (2026-03-15)

## Goal
Allow users to switch map basemap without resetting editor state, while keeping existing drawing, selection, and 3D overlays intact.

## Scope Delivered
- Added `Satellite` / `Streets` toggle in map overlay controls.
- Kept both basemaps in one MapLibre style.
- Switching is visibility-only (`setLayoutProperty`), no `map.setStyle(...)`.
- Satellite is default on first load.
- Streets source uses `https://tile.openstreetmap.org/{z}/{x}/{y}.png`.

## Implementation Notes
- New constants in `MapView/mapViewConstants.ts`:
  - `SATELLITE_SOURCE_ID`, `SATELLITE_LAYER_ID`
  - `STREETS_SOURCE_ID`, `STREETS_LAYER_ID`, `STREETS_SOURCE`
- `createMapStyle()` now registers both basemap sources/layers.
- `MapView.tsx` owns `basemapMode` state and toggles layer visibility after map load.
- `MapOverlayControls.tsx` exposes explicit two-button basemap control.

## Non-Goals / Out Of Scope
- No hybrid satellite+labels mode.
- No runtime style replacement.
- No changes to domain solver behavior.

## Acceptance Criteria
- User can switch between `Satellite` and `Streets`.
- Switching does not clear drawn roofs/obstacles or editing state.
- Orbit/mesh/heatmap behavior remains functional after basemap switches.

## Validation
- Unit: `src/app/features/map-editor/MapView/createMapStyle.test.ts`
  - verifies both basemap sources/layers are present
  - verifies satellite default visibility and streets hidden by default
